# FLOW-007-001

`FLOW-007-001` is the first major revision inside the `FLOW-007` family.

It does not create a new flow family. It hardens the active `FLOW-007` path so the company stops mistaking candidate salvage for process success.

## Why This Revision Exists

Some runs can end with a better artifact after human intervention while still proving that the flow did not autonomously derive the right scope.

The core miss is not "the founder corrected it."

The core miss is:

- target audience was not tightened enough around the real decision context
- domain evidence did not name the real-world decision layer
- the product contract stopped at an intermediate metric
- the flow accepted a mathematically coherent sub-model as if it were complete buyer decision help

## New Rule

If evidence implies a higher real-world decision layer, that higher layer is required scope.

A product is underfit when it:

- models an intermediate metric correctly
- but fails to reach the terminal decision layer the buyer actually optimizes

## Required Contract Changes

`FLOW-007-001` makes the active flow require:

- target-audience lock to capture decision altitude, real-world decision objective, and how success is actually measured
- competitor inspection to extract the benchmark's real-world decision layer, intermediate metrics, terminal metrics, aggregation layer, and misleading partial models
- product architecture contract to define domain evidence in terms of intermediate metrics vs terminal metrics
- decision output contract to classify outputs by layer and require terminal outputs when evidence implies them
- build readiness to fail any scope that solves only an intermediate metric while leaving the buyer's real decision externalized

## Canonical Failure

`REAL_WORLD_DECISION_LAYER_UNDERFIT`

Meaning:

- the flow solved a valid sub-problem
- but not the buyer's actual decision layer
- and no new outside evidence was needed to reveal that gap

## Replay Standard

For this revision class, replay does not pass just because a later artifact looks better.

Replay passes only if, using the original evidence:

- the flow now derives the terminal decision layer itself, or
- the flow blocks the under-scoped candidate before build

## Practical Example

`unit_economy` can be useful and still be incomplete.

If the buyer really decides on total outcome, required unit volume, or another higher-order aggregate, then a unit-economics-only product is under-scoped unless evidence proves that buyers truly act on unit economics alone.
